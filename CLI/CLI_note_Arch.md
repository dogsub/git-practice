# Linux 주요 디렉터리 구조 정리

Linux의 파일 시스템은 최상위 루트 디렉터리 `/`를 기준으로 구성된다.  
`/home`은 최상위 디렉터리가 아니라, `/` 아래에 존재하는 일반 사용자 홈 디렉터리들의 상위 디렉터리이다.

---

## 주요 디렉터리 요약

| 디렉터리 | 설명 |
|---|---|
| `/bin` | 기본 사용자 명령어 실행 파일 |
| `/boot` | 부팅 관련 파일과 커널 |
| `/dev` | 장치 파일 |
| `/etc` | 시스템 설정 파일 |
| `/home` | 일반 사용자 홈 디렉터리 |
| `/lib` | 프로그램 실행에 필요한 라이브러리 |
| `/mnt` | 임시 마운트 지점 |
| `/proc` | 프로세스 및 커널 정보를 제공하는 가상 파일 시스템 |
| `/root` | root 사용자의 홈 디렉터리 |
| `/sbin` | 시스템 관리용 명령어 |
| `/tmp` | 임시 파일 저장 디렉터리 |
| `/usr` | 응용 프로그램, 명령어, 라이브러리, 문서 |
| `/var` | 로그, 캐시 등 자주 변경되는 데이터 |

---

## `/bin`

가장 기본적이고 필수적인 사용자 명령어 실행 파일들이 위치하는 디렉터리이다.

예를 들어 다음과 같은 기본 명령어들이 포함된다.

```bash
ls
cp
mv
rm
cat
mkdir
```

과거에는 `/bin`에 필수 실행 파일들이 직접 존재했지만, 최근 배포판에서는 `/bin`이 `/usr/bin`으로 심볼릭 링크되어 있는 경우도 많다.

확인 예시:

```bash
ls -ld /bin
```

---

## `/boot`

Linux 부팅에 필요한 파일들이 저장되는 디렉터리이다.

주요 구성 요소는 다음과 같다.

```text
커널 이미지
initramfs 또는 initrd
GRUB 부트로더 관련 파일
```

대표적으로 다음과 같은 파일들이 존재할 수 있다.

```text
vmlinuz-*
initramfs-*.img
grub2/
```

주의할 점은 `/boot`는 시스템 부팅과 직접 관련된 디렉터리이므로 임의로 삭제하거나 수정하면 부팅 장애가 발생할 수 있다.

---

## `/dev`

장치 파일들이 위치하는 디렉터리이다.

Linux에서는 디스크, 터미널, USB, 가상 장치 같은 하드웨어 및 장치들을 파일처럼 다룬다.  
이러한 장치 파일들이 `/dev` 아래에 존재한다.

예시:

```text
/dev/sda        # 디스크 장치
/dev/null       # 데이터를 버리는 특수 장치
/dev/zero       # 0 값을 계속 생성하는 특수 장치
/dev/tty        # 현재 터미널
/dev/pts/0      # SSH 또는 터미널 접속 세션
```

즉, `/dev`는 하드웨어 정보 자체를 저장하는 곳이라기보다는 **장치에 접근하기 위한 장치 파일들이 존재하는 곳**이라고 이해하는 것이 더 정확하다.

---

## `/etc`

시스템 전반의 설정 파일들이 위치하는 디렉터리이다.

사용자 계정, 비밀번호, 서비스 설정, 네트워크 설정 등 중요한 설정 파일들이 많이 존재한다.

예시:

```text
/etc/passwd             # 사용자 계정 정보
/etc/shadow             # 암호화된 비밀번호 정보
/etc/group              # 그룹 정보
/etc/hostname           # 호스트 이름
/etc/hosts              # 호스트 이름과 IP 매핑
/etc/fstab              # 파일 시스템 자동 마운트 설정
/etc/ssh/sshd_config    # SSH 서버 설정
```

서버 접속 시 출력되는 메시지와 관련된 파일도 `/etc`에 있다.

```text
/etc/issue       # 로컬 콘솔 로그인 전 메시지
/etc/issue.net   # SSH 로그인 전 배너 메시지
/etc/motd        # 로그인 성공 후 출력되는 메시지
```

SSH 접속 전에 배너를 표시하려면 `/etc/issue.net`을 수정하고 `/etc/ssh/sshd_config`에 다음 설정이 필요하다.

```text
Banner /etc/issue.net
```

수정 후 SSH 서비스를 재시작해야 적용된다.

```bash
sudo systemctl restart sshd
```

로그인 성공 후 메시지를 바꾸고 싶다면 `/etc/motd`를 수정한다.

```bash
sudo vi /etc/motd
```

## `/home`

일반 사용자들의 홈 디렉터리가 위치하는 디렉터리이다.

예를 들어 `dogsub`이라는 사용자가 있다면 일반적으로 홈 디렉터리는 다음과 같다.

```text
/home/dogsub
```

사용자는 자신의 홈 디렉터리 안에 개인 파일, 설정 파일, 작업 디렉터리 등을 저장한다.

```bash
cd ~
```

위 명령어는 현재 로그인한 사용자의 홈 디렉터리로 이동한다.

예시:

```bash
cd /home/dogsub
```

---

## `/lib`

시스템과 프로그램 실행에 필요한 라이브러리 파일들이 위치하는 디렉터리이다.

라이브러리는 실행 파일이 동작할 때 필요한 공통 기능을 담고 있는 파일이다.

예를 들어 `/bin`, `/sbin`에 있는 명령어들이 실행될 때 필요한 공유 라이브러리들이 `/lib` 또는 `/lib64`에 존재할 수 있다.

최근 배포판에서는 `/lib`가 `/usr/lib`로 연결되어 있는 경우도 있다.

확인 예시:

```bash
ls -ld /lib
```

---

## `/mnt`

파일 시스템을 임시로 마운트할 때 사용하는 디렉터리이다.

예전에는 CD-ROM, 플로피 디스크, 외장 디스크 등을 수동으로 마운트할 때 자주 사용했다.

예시:

```bash
sudo mount /dev/sdb1 /mnt
```

요즘은 USB나 외장 디스크 같은 이동식 장치는 `/run/media/사용자명/장치명` 또는 `/media` 아래에 자동 마운트되는 경우도 많다.

정리하면 `/mnt`는 관리자가 임시 마운트 지점으로 사용하는 디렉터리라고 보면 된다.

---

## `/proc`

현재 실행 중인 프로세스와 커널 정보를 파일 형태로 제공하는 가상 파일 시스템이다.

실제 디스크에 저장된 파일이 아니라, 커널이 실시간으로 만들어 보여주는 정보이다.

예시:

```text
/proc/cpuinfo     # CPU 정보
/proc/meminfo     # 메모리 정보
/proc/uptime      # 시스템 가동 시간
/proc/1           # PID 1번 프로세스 정보
```

프로세스마다 PID 번호를 가진 디렉터리가 생성된다.

예시:

```bash
ls /proc
```

---

## `/root`

root 사용자의 홈 디렉터리이다.

일반 사용자의 홈 디렉터리는 보통 `/home/사용자명` 아래에 생성되지만, root 사용자의 홈 디렉터리는 별도로 `/root`에 존재한다.

```text
/root
```

예를 들어 일반 사용자는 다음과 같은 홈 디렉터리를 가진다.

```text
/home/dogsub
```

하지만 root 사용자는 다음 경로를 홈 디렉터리로 사용한다.

```text
/root
```

보안상 일반 사용자는 기본적으로 `/root` 디렉터리에 접근할 수 없다.

---

## `/sbin`

시스템 관리용 명령어들이 위치하는 디렉터리이다.

`/bin`이 일반 사용자도 자주 사용하는 기본 명령어 중심이라면, `/sbin`은 시스템 관리자가 사용하는 명령어 중심이다.

예시:

```text
reboot
shutdown
fdisk
mkfs
ip
```

과거에는 일반 사용자용 명령어는 `/bin`, 관리자용 명령어는 `/sbin`으로 구분하는 의미가 강했다.  
최근 배포판에서는 `/sbin`도 `/usr/sbin`으로 심볼릭 링크되어 있는 경우가 많다.

확인 예시:

```bash
ls -ld /sbin
```

---

## `/tmp`

임시 파일을 저장하는 디렉터리이다.

프로그램 실행 중 일시적으로 필요한 파일이나 사용자가 잠깐 사용하는 파일들이 저장된다.

```text
/tmp
```

`/tmp`는 보통 모든 사용자가 파일을 생성할 수 있도록 쓰기 권한이 열려 있다.  
다만 다른 사용자의 파일을 함부로 삭제하지 못하도록 Sticky Bit이 설정되어 있다.

권한 확인 예시:

```bash
ls -ld /tmp
```

예시 결과:

```text
drwxrwxrwt. 10 root root 4096 Apr 27 20:00 /tmp
```

여기서 마지막 `t`가 Sticky Bit을 의미한다.

주의할 점은 `/tmp`의 파일은 재부팅이나 정리 정책에 의해 삭제될 수 있으므로 중요한 파일을 저장하면 안 된다.

---

## `/usr`

사용자 응용 프로그램, 명령어, 라이브러리, 문서 등이 위치하는 디렉터리이다.

이름은 `user`에서 유래했지만, 현재는 일반 사용자의 개인 파일을 저장하는 곳이라기보다는 **시스템에서 사용하는 대부분의 응용 프로그램과 관련 파일들이 위치하는 곳**으로 이해하는 것이 좋다.

주요 하위 디렉터리는 다음과 같다.

```text
/usr/bin      # 일반 명령어
/usr/sbin     # 시스템 관리 명령어
/usr/lib      # 라이브러리
/usr/share    # 공유 데이터, 문서, 매뉴얼 등
/usr/local    # 관리자가 직접 설치한 프로그램
```

최근 Linux 배포판에서는 `/bin`, `/sbin`, `/lib` 등이 `/usr` 아래로 통합되는 구조를 많이 사용한다.

---

## `/var`

시스템 운영 중 계속 변하는 데이터들이 저장되는 디렉터리이다.

`var`는 variable의 의미로, 로그 파일, 캐시, 메일, 스풀, 패키지 관리 데이터 등 변경이 자주 발생하는 파일들이 위치한다.

예시:

```text
/var/log       # 시스템 및 서비스 로그
/var/cache     # 캐시 데이터
/var/spool     # 작업 대기열 데이터
/var/tmp       # 비교적 오래 유지되는 임시 파일
/var/lib       # 서비스나 패키지가 사용하는 상태 데이터
```

외부 접속 기록이나 서비스 로그도 보통 `/var/log` 아래에 저장된다.

예시:

```text
/var/log/messages
/var/log/secure
/var/log/audit/audit.log
```

RHEL/Rocky Linux 계열에서는 인증, SSH 접속, sudo 사용 기록 등이 주로 `/var/log/secure`에 기록된다.

확인 예시:

```bash
sudo tail -f /var/log/secure
```
