
# 리눅스 저장 공간의 마법사: LVM 완벽 정리

## 1. LVM(Logical Volume Manager)이란?
LVM은 물리적인 디스크 용량을 논리적인 단위로 할당하고 관리하는 **커널의 저장 장치 관리 계층**입니다. 

* **기존 방식 (Standard Partition):** 디스크를 벽돌처럼 딱딱하게 나누는 방식. 한 번 정해진 크기를 변경하려면 데이터를 백업하고 파티션을 삭제해야 하는 번거로움이 있음.
* **LVM 방식:** 디스크를 찰흙처럼 말랑말랑하게 관리하는 방식. 여러 디스크를 하나로 묶거나, 운영 중에도 용량을 자유롭게 늘릴 수 있음.



---

## 2. 왜 LVM을 사용하는가? (필요성)
1.  **유연한 용량 조절:** 서버를 중단하지 않고도 부족한 저장 공간을 확장할 수 있음 (Online Resizing).
2.  **디스크 스트라이핑:** 여러 디스크에 데이터를 분산 저장하여 성능을 높일 수 있음.
3.  **장치 통합:** 500GB 디스크 2개를 묶어 1TB짜리 거대한 가상 디스크 하나로 사용 가능.
4.  **스냅샷:** 특정 시점의 데이터를 보존하여 백업 및 복구에 활용 가능.

---

## 3. LVM의 3단계 구성 요소와 비유

LVM을 이해하기 위해서는 **PV -> VG -> LV**로 이어지는 계층 구조를 반드시 알아야 합니다.

| 용어 | 풀네임 | 비유 | 설명 |
| :--- | :--- | :--- | :--- |
| **PV** | Physical Volume | **식재료 (밀가루)** | 실제 물리 디스크 파티션 (`/dev/nvme...`)을 LVM에서 쓸 수 있게 변환한 상태. |
| **VG** | Volume Group | **반죽 덩어리 (풀)** | 여러 PV를 하나로 합친 거대한 저장 공간의 집합체. |
| **LV** | Logical Volume | **떼어낸 빵 (최종 공간)** | VG에서 사용자가 필요한 만큼만 떼어내어 실제 파일 시스템을 올리는 논리적 공간. |



---

## 4. LVM 구축 실습 흐름 (RHEL 9 기준)

VMware Fusion 환경에서 추가한 새 디스크(예: `/dev/nvme0n2`)를 활용하는 시나리오입니다.

### Step 1. 물리 볼륨(PV) 생성
물리 파티션을 LVM 관리 대상으로 등록합니다.
```bash
# 파티션을 PV로 초기화
sudo pvcreate /dev/nvme0n2p2

# PV 상태 확인 (간략히)
sudo pvs
```

### Step 2. 볼륨 그룹(VG) 생성
생성한 PV들을 하나로 묶어 '자원 풀'을 만듭니다.
```bash
# vgdata라는 이름의 그룹 생성
sudo vgcreate vgdata /dev/nvme0n2p2

# VG 상태 확인
sudo vgs
```

### Step 3. 논리 볼륨(LV) 생성
VG 창고에서 실제로 사용할 만큼의 크기를 떼어냅니다.
```bash
# vgdata에서 5GB를 할당받아 lvdata라는 이름의 LV 생성
sudo lvcreate -L 5G -n lvdata vgdata

# LV 상태 확인
sudo lvs
```

## 5. 구조 확인 및 확장 명령어

### 전체 구조 확인
현재 시스템의 물리 장치와 논리 장치의 관계를 한눈에 확인합니다.
```bash
lsblk
```

### 논리 볼륨 확장 (LVM의 꽃)
사용 중인 공간이 부족할 때 데이터를 유지하며 용량을 늘립니다.
```bash
# lvdata에 2GB를 추가로 더함
sudo lvextend -L +2G /dev/vgdata/lvdata

# 이후 파일시스템 확장 작업이 별도로 필요함 (예: xfs_growfs)
```

## 6. 초보자가 자주 헷갈리는 핵심 체크포인트

1.  **LVM은 파일 시스템인가?**
    * **아니오.** LVM은 저장 공간을 관리하는 '계층'일 뿐입니다. LV를 만든 후에는 반드시 그 위에 `mkfs` 명령어로 파일 시스템(XFS, EXT4 등)을 만들어야 데이터를 저장할 수 있습니다.
2.  **장치 경로는 어디인가?**
    * LV는 보통 `/dev/mapper/vg이름-lv이름` 또는 `/dev/vg이름/lv이름` 경로로 인식됩니다.
3.  **순서가 뒤바뀌면 안 되나?**
    * 반드시 **PV 생성 -> VG 생성 -> LV 생성** 순서를 지켜야 합니다. 재료(PV)가 있어야 반죽(VG)을 만들고, 반죽이 있어야 빵(LV)을 뗄 수 있기 때문입니다.

---

## 7. 요약 및 다음 단계
* **LVM**은 물리적 한계를 극복하기 위한 **가변형 저장소 관리 기술**이다.
* **PV(물리) → VG(그룹) → LV(논리)**의 3단계 구조를 가진다.
* **확장성**이 가장 큰 장점이며, `lvextend`로 온라인 확장이 가능하다.
* **다음 단계:** 생성된 LV를 포맷(파일 시스템 생성)하고 특정 폴더에 연결(마운트)하여 실제 사용을 시작한다.
